## Overview

Technical and minimal with a strong developer focus. The design feels like a sophisticated terminal interface with clean typography, generous whitespace, and a monochromatic aesthetic that emphasizes functionality over decoration. The overall tone is professional yet approachable for technical users.

Content is organized in a single-column layout with consistent left alignment. The design uses a clear grid system based on 8px spacing increments. Sections are separated by generous whitespace rather than visual dividers. The leaderboard uses a clean table layout with consistent row heights and padding.

Uses a 8px base grid with scale: 0.25, 0.5, 0.75, 1, 1.5, 2, 3, 4, 8.

## Typography
- **Headline Font**: Geist
- **Body Font**: Geist
- **Label Font**: Geist
- **Code Font**: Geist Mono

The typography system relies heavily on the Geist font family for both display and body text, with Geist Mono serving all code and terminal-related content. Font weights are used sparingly - primarily 400 for body text and 600 for emphasis. Larger headings use negative letter-spacing for improved readability. The monospace fonts maintain consistent character width for code readability. Size relationships follow a clear hierarchy with body text at 16px and code at 14px for optimal readability.

## Elevation

The design is intentionally flat with minimal use of shadows. Depth is created through subtle border contrasts and background color variations rather than elevation effects. The few shadows present are extremely subtle, primarily used for form elements rather than creating visual hierarchy.

## Components
- **Terminal Command Block**: Dark code block with monospace font, copy button, and terminal-style styling
- **Skills Leaderboard**: Table layout with ranking numbers, skill names, install counts, and consistent row spacing
- **Agent Icons Grid**: Grid of monochromatic icons representing different AI agents and tools
- **Navigation Header**: Clean header with logo, navigation links, and consistent spacing
- **Search Input**: Minimal search input with placeholder text and keyboard shortcut indicator

## Do's and Don'ts
- Do use monospace fonts for all code, commands, and technical content
- Do maintain the 8px spacing grid for consistent alignment
- Do use subtle background variations rather than heavy shadows for depth
- Don't mix serif fonts with the clean sans-serif aesthetic
- Don't use heavy drop shadows or elevation effects
- Do ensure sufficient contrast between text and background colors
- Do use font weight sparingly - primarily 400 and 600
- Don't break the minimal aesthetic with decorative elements
